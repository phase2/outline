        <outline-heading level="h2" levelsize="2xl">
          Lightning fast build times.
        </outline-heading>
        <p>
          Tailwind can take 3–8s to initially compile using our CLI, and upwards
          of 30–45s in webpack projects because webpack struggles with large CSS
          files. This library can compile even the biggest projects in about 800ms
          (with incremental rebuilds as fast as 3ms), no matter what build tool
          you're using.
        </p>
        <outline-heading level="h2" levelsize="2xl">
          Every variant is enabled out of the box.
        </outline-heading>
        <p>
          Variants like focus-visible, active, disabled, and others are not
          normally enabled by default due to file-size considerations. Since this
          library generates styles on demand, you can use any variant you want,
          whenever you want. You can even stack them like
          sm:hover:active:disabled:opacity-75. Never configure your variants
          again.
        </p>
        <outline-heading level="h2" levelsize="2xl">
          Generate arbitrary styles without writing custom CSS.
        </outline-heading>
        <p>
          Ever needed some ultra-specific value that wasn't part of your design
          system, like top: -113px for a quirky background image? Since styles are
          generated on demand, you can just generate a utility for this as needed
          using square bracket notation like top-[-113px]. Works with variants
          too, like md:top-[-113px].
        </p>
        <outline-heading level="h2" levelsize="2xl">
          Your CSS is identical in development and production.
        </outline-heading>
        <p>
          Since styles are generated as they are needed, you don't need to purge
          unused styles for production, which means you see the exact same CSS in
          all environments. Never worry about accidentally purging an important
          style in production again.
        </p>
        <outline-heading level="h2" levelsize="2xl">
          Better browser performance in development.
        </outline-heading>
        <p>
          Since development builds are as small as production builds, the browser
          doesn't have to parse and manage multiple megabytes of pre-generated
          CSS. In projects with heavily extended configurations this makes dev
          tools a lot more responsive.
        </p>
